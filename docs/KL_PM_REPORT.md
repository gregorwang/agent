# KL.md 测试输出评估报告（产品经理 / 问题解决专家视角）

## 1. 目的与范围
本报告针对 `kl.md` 中的“借贷风险评估”输出进行审视，重点评估：
1) 证据是否聚焦“借贷行为本身”；  
2) 话题/关键词召回是否造成噪声；  
3) 结论是否被证据支撑；  
4) 用户信任与可解释性是否成立。

## 2. 执行摘要（结论先行）
- **核心问题**：输出聚焦“借贷”，但证据主体偏向“工资/职业/评价”等外围信息，导致结论“看似合理、实际证据不足”。  
- **关键风险**：话题标签出现“历史事件/历史人物/历史典故/微信用途”等泛化标签，说明话题识别策略存在“为了凑数量而泛化”的倾向，污染检索结果。  
- **用户信任损伤点**：关键词被展示为“命中依据”，但用户无法看到其实际检索权重或命中路径，形成“形式主义”的体验。  
结论：**需要以“借贷证据优先 + 主体归因强制 + 话题白名单约束”重塑检索与输出结构。**

## 3. 用户视角的关键问题
### 3.1 证据偏移
输出结论涉及“是否借钱”，但实际证据更多来自“工资/职业/评价”等周边话题，未能回答最重要的问题：
- 有无明确借款记录  
- 有无逾期或拖延记录  
- 借钱用途与还款习惯  

用户期待的是“借贷风险证据”，而不是“职业背景描述”。这会导致“结论看上去合理，但没有证据支撑”的体验。

### 3.2 话题标签污染
出现“历史事件/历史人物/历史典故/微信用途”等话题，**与借贷问题不相关**。  
这说明话题识别阶段为了凑到最小话题数量，补全了“通用标签”。对用户来说这属于“模型胡扯”或“强行凑数”，会降低信任度。

### 3.3 主体混淆
示例对话讨论的是“高峰”，但被纳入“冯天奇”分析。  
对用户而言这是“逻辑硬伤”，即使模型后续结论正确，也会因这一处证据错误而被否定。

### 3.4 “关键词”呈现与真实检索的错位
系统输出“关键词列表”，但检索链路主要依赖话题索引与语义召回。  
如果用户以为“关键词=命中依据”，就会质疑“关键词看着对，但结果不对”。

## 4. 根因分析（微观到宏观）
### 4.1 微观层面：召回策略缺乏场景约束
借贷问题本应优先检索：
- 借款行为  
- 还款行为  
- 信用 / 逾期 / 欠款  

但检索被扩展到“职业、工资、评价”，形成旁支证据链。

### 4.2 中观层面：话题扩展规则不透明
话题扩展包含“借贷相关 + 通用标签 + 补数量”混合逻辑，导致无关话题进入召回池。

### 4.3 宏观层面：输出结构偏“总结”，弱“证据”
当前输出给出结论，但没有建立“借贷证据清单 → 证据强度评级 → 结论”的结构。

## 5. 产品层面改进建议
### 5.1 证据优先级改造
强制输出结构：
1) 借贷证据清单  
2) 借贷证据不足提示  
3) 仅在证据充分时给出结论  

### 5.2 话题白名单约束
在借贷场景下，话题必须来自：
- 借贷 / 还款 / 欠款 / 信用  
- 工资 / 收入 / 消费习惯  

历史、典故、微信用途等话题直接剔除。

### 5.3 主体归因强制门槛
凡涉及“职业/收入/评价”，必须明确归因到目标人物，否则自动剔除。

### 5.4 可解释性增强
日志输出需包含：
- 语义命中条数  
- 过滤掉多少条  
- 被过滤的原因简述  

用户才能理解“匹配数”“加载数”的关系。

## 6. 流程图（建议版本）
```text
用户问题
  ↓
场景识别（借贷）
  ↓
话题白名单约束 + 关键词扩展
  ↓
话题索引召回 + 语义召回
  ↓
主体归因过滤（非目标人物剔除）
  ↓
证据分层（借贷证据优先）
  ↓
证据不足提示 / 有证据再结论
```

## 7. 最终判断
当前输出“表面合理，但核心证据不足”，容易导致用户对系统产生“只会凑词”的印象。  
正确的方向是：**借贷证据为主干、话题约束为护栏、主体归因为刹车。**
